给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。

请必须使用时间复杂度为 O(log n) 的算法。

两种方法
1.标准二分查找解法
2.Java 内置工具类解法Arrays.binarySearch()
binarySearch(object[] a,object key)


import java.util.Arrays;
class Solution {
    public int searchInsert(int[] nums, int target) {
        int index = Arrays.binarySearch(nums, target);
        
        if (index >= 0) {
            return index;
        } else {
            return -(index + 1);
        }
    }
}
如果target在数组中不存在
index = -(insertion\_point) - 1
insertion_point 是数学概念：它是指 target 本该插入的位置（即第一个比它大的元素的索引）。


源码
// 这是 Arrays 类中的入口方法
public static int binarySearch(int[] a, int key) {
    return binarySearch0(a, 0, a.length, key);
}

// 这是真正执行逻辑的私有方法
private static int binarySearch0(int[] a, int fromIndex, int toIndex, int key) {
    int low = fromIndex;
    int high = toIndex - 1;

    while (low <= high) {
        // 大厂细节 1：使用无符号右移 >>> 1 防止 (low + high) 溢出
        int mid = (low + high) >>> 1;
        int midVal = a[mid];

        if (midVal < key)
            low = mid + 1;
        else if (midVal > key)
            high = mid - 1;
        else
            return mid; // 找到了：返回索引（正数或 0）
    }
    
    // 大厂细节 2：这就是我们讨论的精髓
    // 此时 low 停在的位置，就是 target 本该插入的位置（insertion_point）
    return -(low + 1);  // 没找到：严格遵循 -(insertion_point + 1)
}


把代码跟源码结合在一起就是，我设置一个index接收源码返回的值，在源码里面运行之后返回mid或-（low+1）;当源码运行完返回mid表示存在这个值，并返回索引，且mid>=0，所以回到方法里就是index>=0，方法直接返回index，当源码运行完返回-(low + 1),，表示数组没有这个值，回到方法里就是index<0，返回的 -(index + 1)就是 -(-(low + 1) + 1);

0 的歧义处理，正负号进行“逻辑分流”的做法，无符号右移能保证中点永远是正数，取反逻辑。



1. 原子性与性能（拒绝“扫两次”）在底层的二分查找算法中，“判断是否存在”和“计算插入位置”其实是同一个过程。如果你拆成两个方法：调用 exists()：算法跑一遍，告诉你“不在”。调用 getInsertionPoint()：算法又跑一遍，告诉你“该插在索引 2”。这在处理大规模数据（比如几百万个订单号）时，性能直接下降了一倍。Java 的做法：只跑一遍算法。如果没找到，算法结束时，那个 left 指针刚好就指在插入点上。Java 直接把这个值“打包”成负数丢给你，一次计算，满分效率。
2. 避免“并发修改”导致的逻辑崩溃这在 Java Web 开发或多线程环境下非常关键。假设你先调用 exists(nums, 5) 发现不在，准备再调用 getInsertionPoint(nums, 5)。但在极其微小的瞬间，另一个线程往数组里插了一个数。这时，你第二次调用拿到的“插入点”可能已经失效了。这种“一个返回值给所有答案”的设计，保证了信息的同步性： 既然这个负数是刚才那次搜索的结果，它给出的插入位置就是绝对可靠的。
3. 编程语言的“极简主义”Java 标准库（JDK）在早期设计时，非常推崇**“一个方法解决一类问题”**。如果为每一个需求都设计专门的方法：searchIndex()searchInsertionPoint()searchExists()这会让 API 文档变得极其臃肿。通过公式 $index = -(insertion\_point) - 1$，Java 实现了一种“信息压缩”。
